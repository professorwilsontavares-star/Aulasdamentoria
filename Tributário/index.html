Bug
user faltando no useCallback de salvar
Stale closure — pode salvar com user desatualizado
A função salvar usa user internamente, mas não o inclui nas dependências do useCallback:
const salvar = useCallback(async () => { ... if (!user) { setFormErr("..."); return; } await supabase.from("respostas").insert([{ user_id: user.id, ... }]); }, [form, modal, timerRunning, timerStartedAt]); // ← user ausente!
O useCallback captura o user do momento da criação (inicialmente null). O guard if (!user) protege na maioria dos casos, mas o closure fica tecnicamente obsoleto após o init() autenticar.
// Correto — adicionar user às deps: }, [form, modal, timerRunning, timerStartedAt, user]);

Insert sem verificação de erro
Estado local atualiza mesmo se o Supabase falhar
Em salvar e no botão "Revisei", o resultado do insert é ignorado:
await supabase.from("respostas").insert([...]); setRespostas(prev => [...prev, { ... }]); // roda sempre!

const { error } = await supabase.from("respostas").insert([...]); if (error) { setFormErr("Erro ao salvar. Verifique sua conexão."); return; // não atualiza o estado local } setRespostas(prev => [...prev, novaResposta]);
Atenção
Nav pill branca no dark mode
background hardcoded rgba(255,255,255,.72) não respeita tema escuro
O PillNav recebe darkMode como prop, mas o background da nav usa branco fixo:
background: "rgba(255,255,255,.72)" // sempre branco
Em dark mode, a nav fica branca sobre fundo #0F172A. Contraste quebrado.
background: darkMode ? "rgba(15,23,42,.85)" : "rgba(255,255,255,.72)"
Atenção
tempo_min ausente no useState inicial
form.tempo_min é undefined até o modal ser aberto
O estado inicial do form não inclui tempo_min:
const [form, setForm] = useState({ qtd_questoes: "", qtd_acertos: "", link_erros: "" // tempo_min ausente });
Na prática não causa problema porque o modal sempre chama setForm com tempo_min antes de permitir salvar. Mas é uma inconsistência que pode confundir.
const [form, setForm] = useState({ qtd_questoes: "", qtd_acertos: "", link_erros: "", tempo_min: "" });
Melhoria
sbd e historico usam busca linear O(n²)
metasBanco.find() para cada resposta — escala mal
Em vários lugares, para cada resposta é feita uma busca linear no array de metas:
const m = metasBanco.find(x => x.id === r.meta_id); // O(n) por resp
Com 100 respostas e 500 metas = 50.000 comparações por render. Usar um Map:
// Como useMemo, calculado uma vez: const metasById = useMemo( () => new Map(metasBanco.map(m => [m.id, m])), [metasBanco] ); // Substituir .find() por: const m = metasById.get(r.meta_id);
